TCP和UDP的区别，TCP为什么是三次握手，而不是两次

TCP Transffer Control Protocol 是一种面向连接的，可靠的传输层通信协议。
特点：好比是打电话，是面向连接的，点对点的通信，高可靠的，效率比较低，占用的资源比较多
UDP User Datagram Protocol 是一种无连接的，不可靠的传输层通信协议。
特点：好比是广播：不需要连接，发送方不管接收方有没有准备好，直接发送消息：可以进行广播发送到：传输不可靠，有可能丢失消息，效率比较高；协议就会比较简单，占用的系统资源就会比较少。

TCP建立连接三次握手，断开连接四次挥手。
如果是两次握手，可能会造成链接资源浪费的情况。

JAVA有哪几种IO模型，有什么区别

  • BIO 同步阻塞IO 可靠性差，吞吐量低，适用于连接比较少且比较固定的场景。JDK1.4之前的唯一选择

  • NIO 同步非阻塞IO 可靠性比较好m，吞吐量也比较高，适用于连接比较多并且连接比较短，例如聊天室JDK6开始编程模型复杂

  • AIO 异步非阻塞IO 可靠性最好，吞吐量非常高，适用于连接比较多，并且连接比较长的重操作，例如相册服务器JDK7开始。编程模型比较简单，需要操作系统来支持

JAVA NIO的几个核心组件是什么，分别有什么作用

Channel Buffer Selector

channel类似于流 每个channel对应一个buffer缓冲区。channel会注册到selector。

selector会根据channel上发生的读写事件，将请求交由某个空闲的线程处理。selector对应一个或者多个线程。

buffer和channel都是可读可写的。

HTTP和HTTPS的区别

HTTP是互联网上应用最为广泛的一种网络通信协议，基于TCP，可以使用浏览器工作更为高效，减少网络传输。

HTTPS是HTTP的加强版，可以认为是HTTP+SSL(Secure Socket Layer)。在HTTP的基础上增加了一系列的安全机制一方面保证数据传输安全，另一方面对访问者增加了验证机制，是目前现行架构下，最为安全的解决方案。

  • 主要区别：

    1. HTTP的连接是简单无状态的，HTTPS的数据传输是经过证书加密的，安全性更高
    2. HTTP是免费的，而HTTPS收费
    3. 他们的传输协议不通过，所以他们使用的端口也是不一样的，HTTP默认是80端口，HTTPS是443端口

  • HTTPS的缺点：

    1. HTTPS的握手协议比较费时，所以会影响服务的响应速度以及吞吐量。
    2. HTTPS也并不是完全安全的，他的证书体系其实并不是完全安全的，并且HTTPS在面对DDOS这样的攻击时，几乎起不到任何作用。
